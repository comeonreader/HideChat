# 测试与验收标准 v1.1

适用范围：

* 当前版本范围：邮箱登录、幸运数字伪装入口、联系人、1V1 会话、文本 / 图片 / 文件消息、PIN 解锁本地历史消息

---

# 1. 总体验收原则

必须满足：

* PIN、历史消息不明文落地
* `luckyCode` 允许作为后端校验例外项明文存储
* 会话列表真实内容脱敏
* 搜索结果不返回邮箱
* 图片与通用文件附件链路可用
* 移动端与 PC 端主流程可用

---

# 2. 伪装入口模块

## 2.1 运势页展示

测试点：

* 首页默认展示伪装内容
* 手机和 PC 均正常展示
* 文案接口异常时前端有兜底
* 好运指数、建议列表、情绪提醒、关键词可正确渲染

通过标准：

* 页面无明显聊天系统暴露痕迹

## 2.2 幸运数字校验

测试点：

* 输入数字后调用后端校验接口
* 数据库配置正确时，命中 `luckyCode` 可进入隐藏入口
* 未命中 `luckyCode` 时仍停留在伪装链路
* 数据库缺失有效 `luckyCode` 配置时返回可识别异常
* 不提示“密码错误”字样

通过标准：

* 前端不做浏览器本地幸运数字校验
* 运势页展示的 `luckyNumber` 为用户输入值，且不等同于后端 `luckyCode` 定义

# 3. 用户与联系人模块

## 3.1 获取当前用户信息

测试点：

* 已登录用户获取信息成功
* 未登录访问返回 401
* 返回 `userUid / displayUserId / nickname / avatarUrl`

## 3.2 搜索用户

测试点：

* 可按昵称搜索
* 可按用户 ID 搜索
* 搜索结果不包含邮箱
* 返回 `matchType`
* 返回 `alreadyAdded`
* 高频搜索被限流

通过标准：

* 搜索结果正确
* 无敏感字段泄露

## 3.3 添加联系人

测试点：

* 添加有效联系人成功
* 重复添加幂等
* 不能添加自己
* 不存在用户添加失败

## 3.4 联系人列表

测试点：

* 正确展示已添加联系人
* 按最近消息时间排序
* 无最近消息时排序策略稳定

## 3.5 最近添加列表

测试点：

* 最近添加联系人可见
* 按 `created_at desc` 排序
* 新增联系人后列表刷新正确

# 4. 会话模块

## 4.1 创建或获取 1V1 会话

测试点：

* 与联系人发起聊天时创建会话
* 已存在会话返回原会话
* A-B 和 B-A 不重复创建

## 4.2 会话列表

测试点：

* 能正确展示聊过的用户
* 最后消息时间正确
* 未读数正确
* 在线状态摘要正确
* 不返回本地 PIN 材料

## 4.3 会话列表脱敏规则

测试点：

* 文本消息只显示 `[文本消息]`
* 图片消息只显示 `[图片消息]`
* 文件消息只显示 `[文件消息]`
* 不展示真实消息文本
* 不展示真实文件名

通过标准：

* 列表无敏感内容泄露

## 4.4 清空未读

测试点：

* 打开会话后清空未读成功
* 未读数更新正确

# 6. 本地 PIN 解锁与历史消息

## 6.1 首次设置 PIN

测试点：

* 首次进入会话可设置 PIN
* 不保存 PIN 明文
* 后续可用该 PIN 解锁会话

## 6.2 PIN 解锁会话

测试点：

* 输入正确 PIN 可解锁历史消息
* 输入错误 PIN 提示失败
* 错误 PIN 不泄露更多信息

## 6.3 锁定恢复

测试点：

* 页面刷新后需重新输入 PIN
* 切后台或超时后自动锁定
* 未输入 PIN 时只能看到会话索引，不能看到消息明文

# 7. 文件与安全测试

## 7.1 文件安全

测试点：

* 非法文件类型上传拦截
* 超大文件拦截
* 非授权访问文件信息拦截
* 非会话参与方不可获取下载地址

## 7.2 搜索安全

测试点：

* 用户搜索频控有效
* 搜索结果不返回邮箱
* 搜索输入异常字符不导致越权或报错

## 7.3 本地存储安全

测试点：

* `luckyCode` 仅允许在后端数据库中作为校验例外项明文存储
* PIN 是否明文存储
* 消息与附件是否以密文方式缓存

# 9. 必须通过项

| 项目 | 标准 | 等级 |
| --- | --- | --- |
| 幸运数字入口 | 正确进入，错误保持伪装 | 必须 |
| 搜索用户 | 支持昵称、用户 ID，且不返回邮箱 | 必须 |
| 最近添加列表 | 可展示且排序正确 | 必须 |
| 会话列表脱敏 | 不暴露真实文本和文件名 | 必须 |
| 图片消息 | 可上传、发送、预览 | 必须 |
| 文件消息 | 可上传、发送、下载，支持可预览类型预览 | 必须 |
| PIN 解锁 | 正确 PIN 可解密历史 | 必须 |
| 未读数 | 基本准确 | 必须 |
| 移动端布局 | 聊天列表 / 聊天页 / 搜索页可用 | 必须 |
| 敏感信息存储 | `luckyCode` 可明文存后端库，其余 PIN、历史消息不得明文落地 | 必须 |
