# 数据库字段设计文档 v1.2

适用范围：

* 后端：Spring Boot + PostgreSQL
* 前端：Web 浏览器本地持久化（IndexedDB）
* 场景：1V1 聊天、邮箱登录、幸运数字伪装入口、直接进入聊天、图片与通用文件附件

---

## 1. 文档目标

本文档定义本系统的：

* 后端数据模型
* 前端本地数据模型
* 字段设计与索引建议
* 隐私边界

---

## 2. 设计原则

### 2.1 后端与前端职责分离

后端负责：

* 用户账号
* 联系人关系
* 会话索引
* 消息中转与存档
* 通用附件元数据
* 未读计数

前端负责：

* 会话列表渲染
* 本地必要消息缓存
* 刷新恢复

### 2.2 明确不落地的数据

以下数据不放后端数据库，也不要求放前端本地库：

* PIN 明文
* `pinVerifierHash`
* `pinSalt`
* `pinKdfParams`
* `encryptedConversationKey`
* `conversationKey`

### 2.3 列表脱敏原则

会话列表只存和返回脱敏预览，不存真实消息文本与真实文件名。

## 5. 关系与约束汇总

| 主表 | 关系 | 从表 | 说明 |
| --- | --- | --- | --- |
| im_user | 1:N | im_contact | 一个用户拥有多个联系人视图 |
| im_user | 1:N | im_message | 一个用户可发送 / 接收多条消息 |
| im_user | 1:N | im_file | 一个用户可上传多个附件 |
| im_conversation | 1:N | im_message | 一个会话包含多条消息 |
| im_message | 0..1:1 | im_file | 图片 / 文件消息关联附件 |
| im_user + im_conversation | 1:1 | im_unread_counter | 每用户对每会话一条未读记录 |

---

## 6. 安全与隐私说明

* 不记录 PIN 明文
* `luckyCode` 允许在后端数据库明文存储，用于后端校验
* 会话列表不记录真实消息预览
* 搜索结果不以邮箱作为展示字段
* 本地缓存结构不包含 PIN 校验材料
