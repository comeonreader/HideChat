# 后端接口定义文档 v1.2

适用范围：

* 后端：Spring Boot 3.x
* 认证：邮箱登录 + JWT + Refresh Token
* 通信：REST API + WebSocket
* 当前范围：MVP / V1.2
* 场景：Web 端 1V1 聊天、联系人、会话列表、图片与通用文件附件、伪装入口

---

## 1. 文档目标

本文档定义系统后端 REST 与 WebSocket 接口：

* 接口分组
* 请求 / 响应规范
* 字段定义
* 鉴权与错误码
* 与前端本地缓存边界

说明：

* 幸运数字校验通过后端接口完成
* 认证成功后前端直接进入聊天系统
* 后端不提供独立 PIN API / Service / 模型

---

## 2. 通用约定

### 2.1 前缀

```text
/api
```

### 2.2 认证方式

除注册、登录、发送验证码、重置密码、获取伪装页配置外，其余接口默认需要：

```http
Authorization: Bearer <accessToken>
```

### 2.3 统一响应结构

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 2.4 时间格式

* 对外统一返回毫秒时间戳 `long`

## 3. 错误码

通用错误码：

| 错误码 | 含义 |
| --- | --- |
| 0 | 成功 |
| 400001 | 参数错误 |
| 401001 | 未登录或 token 无效 |
| 403001 | 无权限访问 |
| 404001 | 资源不存在 |
| 409001 | 资源已存在 |
| 429001 | 请求过于频繁 |
| 500001 | 系统内部错误 |
| 500003 | 文件服务异常 |

业务错误码：

| 错误码 | 含义 |
| --- | --- |
| 410101 | 邮箱已注册 |
| 410102 | 用户不存在 |
| 410103 | 密码错误 |
| 410104 | 邮箱验证码无效 |
| 420101 | 联系人不存在 |
| 420102 | 会话不存在 |
| 420103 | 消息不存在 |
| 420104 | 文件不存在 |
| 420105 | 不支持的消息类型 |
| 420201 | luckyCode 校验失败 |
| 420202 | luckyCode 数据库配置缺失 |

---

## 5. Auth 模块

### 5.1 发送邮箱验证码

`POST /api/auth/email/send-code`

### 5.2 邮箱注册

`POST /api/auth/email/register`

### 5.3 邮箱密码登录

`POST /api/auth/email/password-login`

### 5.4 邮箱验证码登录

`POST /api/auth/email/code-login`

### 5.5 重置密码

`POST /api/auth/email/reset-password`

### 5.6 刷新 token

`POST /api/auth/refresh-token`

### 5.7 退出登录

`POST /api/auth/logout`

说明：

* 登录成功后前端可直接进入聊天系统
* 后端不返回 PIN、会话密钥或解锁材料

---

## 6. User 模块

### 6.1 获取当前用户信息

`GET /api/user/me`

响应示例：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "userUid": "u_1001",
    "displayUserId": "hide_1001",
    "nickname": "Alice",
    "avatarUrl": ""
  }
}
```

### 6.2 搜索用户

`GET /api/user/search?keyword=hide_1077`

说明：

* 支持按昵称搜索
* 支持按展示用用户 ID 搜索
* 不支持按邮箱搜索

---

## 7. Contact 模块

### 7.1 添加联系人

`POST /api/contact/add`

### 7.2 获取联系人列表

`GET /api/contact/list`

说明：

* 默认按最近消息时间倒序
* 如无最近消息，则按联系人创建时间倒序补位

### 7.3 获取最近添加联系人

`GET /api/contact/recent?limit=10`

### 7.4 更新联系人备注 / 置顶状态

`PUT /api/contact/update`

---

## 8. Conversation 模块

### 8.1 创建或获取 1V1 会话

`POST /api/conversation/single`

### 8.2 获取会话列表

`GET /api/conversation/list`

说明：

* 返回脱敏预览
* 返回未读数和最近消息时间
* 不返回 PIN 相关字段
* 不返回本地解锁材料

### 8.3 清空未读

`POST /api/conversation/clear-unread`

---

## 9. Message 模块

### 9.1 发送消息

`POST /api/message/send`

支持：

* `text`
* `image`
* `file`

### 9.2 拉取历史消息

`GET /api/message/history`

说明：

* 支持分页
* 必须校验会话归属
* 历史消息由前端直接展示，不再经过 PIN 解锁流程

### 9.3 已读回执

`POST /api/message/read`

---

## 10. File 模块

### 10.1 获取上传签名

`POST /api/file/upload-sign`

### 10.2 上传对象

`PUT /api/file/upload/{fileId}`

### 10.3 上传完成回调

`POST /api/file/complete`

### 10.4 获取文件信息

`GET /api/file/{fileId}`

### 10.5 获取文件内容

`GET /api/file/content/{fileId}`

---

## 11. System / Disguise 模块

### 11.1 获取今日运势

`GET /api/system/fortune/today`

### 11.2 获取伪装配置

`GET /api/system/disguise-config`

### 11.3 校验幸运数字

`POST /api/system/disguise/verify-lucky-number`

说明：

* 命中 `luckyCode` 后允许进入认证页或聊天系统
* 未命中时停留在伪装链路

---

## 12. WebSocket 模块

### 12.1 建连

`GET /ws/chat`

### 12.2 客户端上行事件

* `CHAT_SEND`
* `CHAT_READ`

### 12.3 服务端下行事件

* `CHAT_RECEIVE`
* `CHAT_ACK`
* `CHAT_READ`

---

## 13. 前后端边界说明

### 13.1 后端负责

* 账号体系
* 联系人 / 会话 / 消息
* 通用附件
* 未读管理
* 在线状态
* 运势内容
* 幸运数字校验

### 13.2 前端负责

* 幸运数字输入页与运势页展示
* 认证成功后的页面跳转
* 本地 `message_local` 等必要缓存
* 会话列表脱敏渲染

### 13.3 当前版本不采用

* PIN 解锁接口
* PIN 校验材料下发
* PIN -> KEK -> conversationKey 协议

---

## 14. 兼容性要求

* 本轮去除 PIN 解锁流程不涉及接口协议调整
* 本轮不涉及数据库 schema 变更
