# 后端接口定义文档 v1.1

适用范围：

* 后端：Spring Boot 3.x
* 认证：邮箱登录 + JWT + Refresh Token
* 通信：REST API + WebSocket
* 当前范围：MVP / V1.1
* 场景：Web 端 1V1 聊天、联系人、会话列表、图片与通用文件附件、伪装入口

---

# 1. 文档目标

本文档定义系统后端 REST 与 WebSocket 接口：

* 接口分组
* 请求 / 响应规范
* 字段定义
* 鉴权与错误码
* 与前端本地加密边界

说明：

* 幸运数字校验通过后端接口完成
* PIN 解锁本地聊天记录默认在前端完成
* 后端负责账号体系、联系人 / 会话 / 消息、通用附件、未读管理、在线状态、运势内容

---

# 2. 通用约定

## 2.1 前缀

```text
/api
```

## 2.2 认证方式

除注册、登录、发送验证码、重置密码、获取伪装页配置外，其余接口默认需要：

```http
Authorization: Bearer <accessToken>
```

## 2.3 统一响应结构

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

## 2.4 时间格式

* 对外统一返回毫秒时间戳 `long`

# 3. 错误码

通用错误码：

| 错误码 | 含义 |
| --- | --- |
| 0 | 成功 |
| 400001 | 参数错误 |
| 401001 | 未登录或 token 无效 |
| 403001 | 无权限访问 |
| 404001 | 资源不存在 |
| 409001 | 资源已存在 |
| 429001 | 请求过于频繁 |
| 500001 | 系统内部错误 |
| 500003 | 文件服务异常 |

业务错误码：

| 错误码 | 含义 |
| --- | --- |
| 410101 | 邮箱已注册 |
| 410102 | 用户不存在 |
| 410103 | 密码错误 |
| 410104 | 邮箱验证码无效 |
| 420101 | 联系人不存在 |
| 420102 | 会话不存在 |
| 420103 | 消息不存在 |
| 420104 | 文件不存在 |
| 420105 | 不支持的消息类型 |
| 420201 | luckyCode 校验失败 |
| 420202 | luckyCode 数据库配置缺失 |

# 6. User 模块

## 6.1 获取当前用户信息

`GET /api/user/me`

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "userUid": "u_1001",
    "displayUserId": "hide_1001",
    "nickname": "Alice",
    "avatarUrl": ""
  }
}
```

## 6.2 搜索用户

`GET /api/user/search?keyword=hide_1077`

说明：

* 支持按昵称搜索
* 支持按展示用用户 ID 搜索
* 不支持按邮箱搜索

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "userUid": "u_1002",
      "displayUserId": "hide_1077",
      "nickname": "林夏",
      "avatarUrl": "",
      "matchType": "display_user_id",
      "alreadyAdded": false
    }
  ]
}
```

业务规则：

* 不返回邮箱
* 空关键字按产品策略返回空数组或 400001
* 高频搜索需限流

# 7. Contact 模块

## 7.1 添加联系人

`POST /api/contact/add`

请求体：

```json
{
  "peerUid": "u_1002",
  "remarkName": "林夏"
}
```

## 7.2 获取联系人列表

`GET /api/contact/list`

说明：

* 默认按最近消息时间倒序
* 如无最近消息，则按联系人创建时间倒序补位

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "peerUid": "u_1002",
      "displayUserId": "hide_1077",
      "peerNickname": "林夏",
      "peerAvatar": "",
      "remarkName": "林夏",
      "pinned": false,
      "lastMessageAt": 1712300000000
    }
  ]
}
```

## 7.3 获取最近添加联系人

`GET /api/contact/recent?limit=10`

说明：

* 按 `created_at desc` 排序

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "peerUid": "u_1002",
      "displayUserId": "hide_1077",
      "peerNickname": "林夏",
      "peerAvatar": "",
      "createdAt": 1712300000000
    }
  ]
}
```

## 7.4 更新联系人备注 / 置顶状态

`PUT /api/contact/update`

## 7.5 删除联系人

`DELETE /api/contact/{peerUid}`

# 8. Conversation 模块

## 8.1 获取会话列表

`GET /api/conversation/list`

说明：

* 返回用户已存在的 1V1 会话列表
* 不返回本地 PIN 材料
* 不返回本地密文缓存
* 列表预览必须脱敏

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "conversationId": "c_1001",
      "peerUid": "u_1002",
      "displayUserId": "hide_1077",
      "peerNickname": "林夏",
      "peerAvatar": "",
      "remarkName": "林夏",
      "lastMessageType": "file",
      "lastMessagePreview": "[文件消息]",
      "previewStrategy": "masked",
      "lastMessageAt": 1712300000000,
      "unreadCount": 2,
      "pinned": false,
      "onlineStatus": "online",
      "onlineStatusText": "手机在线"
    }
  ]
}
```

约束：

* `lastMessagePreview` 仅允许固定占位：`[文本消息]` / `[图片消息]` / `[文件消息]`
* 不允许返回真实文件名与真实消息文本

## 8.2 获取或创建 1V1 会话

`POST /api/conversation/single`

请求体：

```json
{
  "peerUid": "u_1002"
}
```

## 8.3 获取会话详情

# 9. Message 模块

## 9.1 拉取历史消息

`GET /api/message/history?conversationId=c_1001&cursor=1712300000000&pageSize=20`

说明：

* 前端优先读本地 IndexedDB
* 此接口主要用于首次同步、补偿拉取、重连恢复

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "messageId": "m_1001",
        "conversationId": "c_1001",
        "senderUid": "u_1002",
        "receiverUid": "u_1001",
        "messageType": "file",
        "payloadType": "ref",
        "payload": {
          "fileId": "f_1001"
        },
        "fileId": "f_1001",
        "clientMsgTime": 1712300000000,
        "serverMsgTime": 1712300001000,
        "serverStatus": "delivered"
      }
    ],
    "nextCursor": "1712290000000",
    "hasMore": true
  }
}
```

`messageType` 允许值：

* `text`
* `image`
* `file`
* `system`

## 9.2 标记消息已读

`POST /api/message/read`

请求体：

```json
{
  "conversationId": "c_1001",
  "messageIds": ["m_1001", "m_1002"]
}
```

## 9.3 批量获取补偿消息

`GET /api/message/pending?since=1712300000000`

* 图片属于 `fileCategory=image` 的一种

`POST /api/file/upload-sign`

```json
{
  "fileName": "project-draft-v3.pdf",
  "fileExt": "pdf",
  "fileCategory": "document",
  "mimeType": "application/pdf",
  "fileSize": 2516582,
  "encryptFlag": true
```

```json
{

      "Content-Type": "application/octet-stream"
```

# 11. System / Disguise 模块

## 11.1 获取今日运势内容

`GET /api/system/fortune/today`

说明：

* 仅用于伪装页内容展示
* `luckyNumber` 返回用户输入的数字，用于运势页展示
* `luckyNumber` 不是后端数据库里的 `luckyCode`

请求参数：

| 参数 | 位置 | 必填 | 说明 |
| --- | --- | --- | --- |
| luckyNumber | query | 否 | 用户输入值；传入后原样回显到运势页 |

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "title": "今日运势",
    "summary": "今天适合整理情绪与节奏。",
    "fortuneScore": 86,
    "luckyColor": "青绿色",
    "luckyNumber": "13",
    "suggestedActions": "沟通 / 整理",
    "adviceList": [
      "先处理最重要的事",
      "表达尽量简单清楚",
      "别急着证明自己"
    ],
    "moodReminder": "今天容易在傍晚时分感到疲惫，适合保留安静空间。",
    "keywords": [
      "放松",
      "行动",
      "耐心"
    ]
  }
}
```

业务规则：

* 当前接口不做 `luckyCode` 校验
* 若前端传入 `luckyNumber`，后端按原值返回用于伪装展示
* 若前端未传入 `luckyNumber`，后端可返回默认展示值

## 11.2 校验幸运数字

`POST /api/system/disguise/verify-lucky-number`

说明：

* 前端提交用户输入数字
* 后端读取数据库中的 `luckyCode` 明文进行校验
* 命中时进入隐藏入口，未命中时继续伪装链路

请求体：

```json
{
  "luckyNumber": "13"
}
```

成功响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "matched": false,
    "nextAction": "SHOW_FORTUNE",
    "luckyNumber": "13"
  }
}
```

命中隐藏入口响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "matched": true,
    "nextAction": "ENTER_HIDDEN_ENTRY",
    "luckyNumber": "13"
  }
}
```

失败响应：

参数错误：

```json
{
  "code": 400001,
  "message": "luckyNumber is required",
  "data": null
}
```

数据库未配置有效 luckyCode：

```json
{
  "code": 420202,
  "message": "luckyCode config missing",
  "data": null
}
```

业务规则：

* 接口不返回数据库中的 `luckyCode`
* 校验比较在后端完成，前端不再做本地 hash / 本地比对
* 返回体里的 `luckyNumber` 为用户输入值，供后续运势页展示

## 11.3 获取伪装页配置

`GET /api/system/disguise-config`

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "siteTitle": "今日运势",
    "showFortuneInput": true,
    "theme": "default",
    "mobileBottomNav": true
  }
}
```

# 13. 接口边界说明

前端本地完成：

* PIN 校验
* 本地历史消息解密

后端提供：

* 用户与联系人数据
* 会话列表与脱敏预览
* 消息补偿
* 附件元数据
* luckyCode 校验
* 运势内容
* 在线状态摘要
