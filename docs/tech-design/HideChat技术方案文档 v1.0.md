# 技术方案文档 v1.2

## 项目名称

HideChat 隐私伪装聊天 Web 应用

## 1. 项目背景

本项目目标是构建一个支持手机浏览器与 PC 浏览器的 Web 聊天应用。当前版本通过伪装入口、认证闭环、会话列表脱敏和本地必要缓存，实现“低暴露、可直接使用”的聊天体验。

当前版本能力范围：

* 运势伪装入口
* 幸运数字后端校验
* 会话列表脱敏展示
* 联系人搜索与添加
* 1V1 文本、图片、通用文件附件消息
* 浏览器本地消息缓存
* 在线状态与未读数展示
* 移动端单列布局与底部导航

当前版本明确不采用：

* PIN 设置 / 输入 / 解锁
* PIN -> KEK -> conversationKey
* `encryptedConversationKey`
* 自动锁定后重新认证聊天历史

## 2. 建设目标

### 2.1 产品目标

实现一个“伪装入口 + 认证直达聊天 + 本地必要缓存 + 基础聊天体验”的聊天系统。

### 2.2 技术目标

1. Web 端伪装入口机制
2. 浏览器本地消息缓存
3. 联系人 / 会话搜索与索引能力
4. 文本、图片、通用文件附件消息
5. 在线状态、未读数与补偿拉取
6. 手机与 PC 自适应布局

---

## 3. 非目标范围

1. 不做原生 App
2. 不做群聊
3. 不做多设备密钥同步
4. 不做 Signal 双棘轮
5. 不做真正意义上的强端到端加密体系
6. 不做通讯录导入
7. 不做邮箱搜索用户
8. 不做聊天列表真实消息预览
9. 不做音视频通话
10. 不承诺绝对防破解或防截屏

## 4. 总体架构设计

### 4.1 架构概览

系统采用前后端分离架构：

* 前端负责页面展示、本地缓存、会话搜索过滤、聊天状态编排
* 后端负责账号、联系人、会话索引、消息中转、附件元数据、在线状态、离线补偿、`luckyCode` 校验

```text
┌────────────────────────────────────────────┐
│                  Web 前端                  │
│ 1. 幸运数字输入页                          │
│ 2. 运势展示页                              │
│ 3. 认证页                                  │
│ 4. 聊天列表页                              │
│ 5. 搜索 / 添加好友页                       │
│ 6. 聊天页                                  │
│ 7. IndexedDB 本地消息缓存                  │
│ 8. WebSocket 客户端                        │
└────────────────────┬───────────────────────┘
                     │ HTTPS / WSS
┌────────────────────▼───────────────────────┐
│              Spring Boot 后端              │
│ 1. Auth / User / Contact                   │
│ 2. Conversation / Message                  │
│ 3. File / Attachment                       │
│ 4. System / Disguise                       │
│ 5. WebSocket Gateway                       │
└───────────────┬───────────────┬────────────┘
                │               │
       ┌────────▼──────┐  ┌────▼─────────┐
       │ PostgreSQL    │  │ Redis         │
       │ 用户/联系人/  │  │ 在线状态/会话 │
       │ 会话/附件元数据│ │ 映射/短期缓存 │
       └───────────────┘  └───────────────┘
```

### 4.2 架构原则

#### 4.2.1 前端职责

* 幸运数字仅用于伪装入口
* 认证成功后直接进入聊天
* 会话列表不得显示真实消息明文与真实文件名
* 本地仅保留必要缓存，例如 `message_local`

#### 4.2.2 后端职责

* 管理用户、联系人、会话、附件元数据
* 提供搜索、排序、未读、在线状态
* 不提供独立 PIN API

#### 4.2.3 安全边界

本方案目标是：

* 降低聊天入口暴露
* 防止会话列表直接泄露真实内容

不是：

* 对抗高级取证
* 对抗浏览器被攻陷
* 对抗 root / 越狱 / 远程注入

## 5. 核心业务流程

### 5.1 首次使用

```text
访问首页
→ 展示幸运数字输入页
→ 用户输入幸运数字
→ 调用后端接口校验 luckyCode
→ 未命中 luckyCode：展示运势页
→ 命中 luckyCode：进入认证页
→ 用户注册或登录
→ 拉取联系人、会话、历史消息
→ 进入聊天系统
```

### 5.2 日常进入

```text
访问首页
→ 输入幸运数字
→ 调用后端接口校验 luckyCode
→ 成功后进入认证页或直接进入聊天系统
→ 加载会话列表
→ 点击某个联系人
→ 直接展示聊天页
```

### 5.3 收消息

```text
对方发送消息
→ 服务端接收消息与附件引用
→ 更新会话索引与未读数
→ 若接收方在线则实时推送
→ 前端收到消息
→ 更新当前会话
→ 写入本地消息缓存
```

### 5.4 搜索与添加联系人

```text
用户进入搜索 / 添加好友页
→ 输入昵称或用户 ID
→ 后端搜索用户
→ 返回昵称、展示用用户 ID、匹配类型、已添加状态
→ 用户点击添加
→ 联系人关系落库
→ 最近添加列表刷新
```

### 5.5 返回伪装页

```text
用户点击返回伪装页
→ 关闭聊天页实时连接
→ 返回运势 / 幸运数字入口
→ 不触发 PIN 流程
```

## 7. 模块拆分

* `auth`：登录注册、JWT 鉴权
* `user`：用户资料、按昵称 / 用户 ID 搜索
* `contact`：联系人添加、最近添加列表
* `disguise`：幸运数字输入页配置、运势内容下发
* `conversation`：会话索引、列表脱敏预览、未读计数、排序
* `message`：文本 / 图片 / 文件消息、历史补偿、已读回执
* `file`：通用附件上传、元数据回写、预览 / 下载地址
* `websocket`：在线状态、消息推送、ACK 处理

## 8. 前端状态与交互设计

### 8.1 页面状态模型

前端至少维护以下状态：

* `disguiseState`：幸运数字是否已校验通过
* `fortuneState`：运势内容与伪装配置
* `authState`：认证方式、认证结果、当前用户
* `conversationListState`：会话列表、搜索关键字、排序、未读数
* `contactSearchState`：搜索关键字、搜索结果、添加状态、最近添加列表
* `chatState`：当前会话消息流、输入态、附件上传态
* `onlineState`：对端在线 / 最近活跃摘要

### 8.2 搜索策略

#### 8.2.1 会话搜索

* 仅在前端本地对已加载会话做过滤
* 匹配字段为备注名、昵称、用户 ID
* 不搜索消息明文

#### 8.2.2 联系人搜索

* 走后端接口
* 支持昵称与用户 ID 搜索
* 不支持邮箱搜索

### 8.3 列表脱敏预览策略

会话列表只允许渲染固定占位：

* `[文本消息]`
* `[图片消息]`
* `[文件消息]`

### 8.4 在线状态来源与刷新

* 会话列表初始化时拉取一次
* 当前聊天页优先订阅实时状态更新
* 状态为弱一致展示，不参与核心权限判断

---

## 9. 本地存储设计

### 9.1 幸运数字设计

幸运数字是伪装入口凭证，不承担消息缓存解锁职责。

定义：

* `luckyCode`：后端数据库预置的隐藏入口校验值，明文存储，仅用于后端校验
* `luckyNumber`：用户在伪装页输入的数字，用于运势页展示

### 9.2 本地缓存设计

当前版本仅保留必要本地缓存：

* 会话索引短时缓存
* `message_local` 消息缓存

当前版本不采用：

* PIN 校验材料
* PIN 衍生 KDF 参数
* `encryptedConversationKey`
* `conversationKey` 驻留设计

### 9.3 消息缓存策略

* 会话消息可直接序列化后写入 IndexedDB
* 刷新页面后优先恢复本地缓存
* 本地缓存缺失时回退后端历史接口

## 11. API 设计

### 11.1 用户搜索

`GET /api/user/search?keyword=xxx`

返回字段建议：

* `userUid`
* `displayUserId`
* `nickname`
* `avatarUrl`
* `matchType`
* `alreadyAdded`

限制：

* 不返回邮箱

### 11.2 联系人与会话列表

* 联系人列表按最近消息时间排序
* 最近添加列表按 `created_at desc` 排序
* 会话列表返回脱敏预览、未读数、在线状态摘要

### 11.3 文件接口

* 支持图片与非图片文件
* 返回 `fileName / fileExt / fileCategory / previewable / downloadUrl`

---

## 12. 安全要求

* 禁止日志输出幸运数字、消息明文、token 明文
* `luckyCode` 仅允许后端数据库明文存储
* 会话列表不得返回真实消息预览
* 前端本地缓存结构不得包含 PIN 材料

---

## 13. 错误处理

* 幸运数字校验失败：停留在伪装链路
* 认证失败：停留在认证页并展示错误
* 本地缓存恢复失败：回退后端历史消息接口
* WebSocket 失败：回退 HTTP 消息发送

---

## 14. 验收重点

* 主链路应为 `disguise -> auth -> chat`
* 点击会话直接进入聊天
* 刷新后无需 PIN 仍可恢复使用
* 本地消息缓存不依赖 PIN 解锁
