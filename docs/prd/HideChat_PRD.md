# PRD v1.1：隐私伪装聊天 Web 应用

---

# 1. 产品概述

## 1.1 产品定位

一个支持以下能力的 Web 隐私聊天工具：

* 伪装入口（幸运数字）
* 本地加密聊天（PIN 解锁）
* 浏览器持久化隐私存储
* 联系人搜索与添加
* 1V1 文本、图片、通用文件附件消息

## 1.2 核心能力

当前版本重点：

* 加密存储聊天记录（本地）
* PIN 解锁聊天内容
* 会话列表可见，但真实内容不可见
* 伪装页在手机与 PC 浏览器均可自然展示

## 1.3 核心设计理念

> 让聊天存在，但看不见。

# 2. 用户流程

```text
打开网页
  ↓
展示幸运数字输入页
  ↓
输入幸运数字
  ↓
通过后端接口校验 luckyCode
  ↓
命中 luckyCode：进入聊天系统
未命中 luckyCode：展示运势页
  ↓
展示聊天列表（联系人 / 未读 / 时间 / 脱敏预览）
  ↓
搜索联系人或进入添加好友页
  ↓
点击某个会话
  ↓
输入 PIN code
  ↓
解密聊天记录
  ↓
进入聊天界面并继续发送文本 / 图片 / 文件
```

# 3. 页面信息架构

## 3.1 幸运数字输入页

页面目标：

* 作为伪装入口的第一层
* 输入幸运数字，不暴露“聊天解锁”语义

页面元素：

* 顶部情绪/日期类伪装文案
* 幸运数字输入框
* 进入按钮

交互要求：

* 幸运数字由后台数据库定义 luckyCode 作为隐藏入口校验值
* 用户输入后通过后端接口校验
* 校验失败不提示密码错误，只停留在伪装链路

## 3.2 运势展示页

页面目标：

* 提供完整伪装内容，降低聊天系统暴露感

页面元素：

* 运势标题与摘要
* 好运指数
* 幸运色 / 幸运数字（展示用户输入值，非 luckyCode） / 宜做事项
* 建议列表
* 情绪提醒
* 返回首页或进入聊天系统的后续动作

## 3.3 聊天列表页

页面目标：

* 展示最近会话与联系人入口
* 支持搜索会话
* 支持移动端底部导航

页面元素：

* 顶部标题与“搜索 / 添加好友”入口
* 会话搜索框
* 会话列表
* 未读徽标
* 在线态摘要
* 移动端底部导航

列表展示字段：

* 对端昵称 / 备注名
* 最近消息时间
* 未读数
* 在线态摘要
* 脱敏预览

脱敏预览策略：

* 不允许展示真实消息文本
* 不允许展示真实文件名
* 统一展示固定占位
* 示例：`[文本消息]`、`[图片消息]`、`[文件消息]`

## 3.4 聊天页

页面目标：

* 展示已解锁会话的消息明文
* 支持文本、图片、文件消息发送

页面元素：

* 会话头部
* 在线状态/活跃时间提示
* 消息流
* 文本输入框
* 图片发送按钮
* 文件发送按钮

文件消息展示要求：

* 聊天页内可展示真实文件名、大小、预览/下载动作
* 会话列表不得展示真实文件名

## 3.5 搜索 / 添加好友页

页面目标：

* 支持按昵称、用户 ID 搜索用户
* 支持添加联系人
* 支持展示最近添加列表

页面元素：

* 搜索框
* 搜索结果列表
* 最近添加列表
* 已添加 / 可添加状态

搜索结果展示字段：

* 用户昵称
* 展示用用户 ID
* 是否已添加

展示限制：

* 不返回、不展示用户邮箱

# 4. 功能模块设计

## 4.1 模块一：幸运数字入口

### 4.1.1 功能说明

* 后台数据库预置 luckyCode 作为隐藏入口校验值
* 用户每次访问输入数字后，前端调用后端接口完成校验
* 若输入值未命中 luckyCode，则继续停留在伪装链路并展示运势页

### 4.1.2 存储方式

```text
Backend Database:
im_disguise_lucky_code
- code_value
- status
- effective_start_at
- effective_end_at

Frontend Runtime:
- lastInputLuckyNumber
```

要求：

* luckyCode 明文存于后端数据库，仅用于后端校验
* 运势页 luckyNumber 展示用户本次输入值

### 4.1.3 错误行为

* 不提示错误
* 只刷新或停留在运势内容

## 4.2 模块二：聊天列表

### 4.2.1 数据来源

会话列表来源于：

* 后端会话索引
* 本地会话锁定状态
* 本地未读状态补充

### 4.2.2 会话搜索

* 支持按备注名、昵称、本地已有用户 ID 搜索
* 默认优先前端本地过滤当前已加载列表
* 不检索真实消息明文

### 4.2.3 在线态

* 展示在线 / 最近活跃等摘要信息
* 仅作为体验信息，不作为强一致业务依据

### 4.2.4 未读徽标

* 展示会话级未读数
* 打开会话后清零或递减

### 4.2.5 脱敏策略

* 列表仅显示类型占位
* 不暴露文本明文
* 不暴露文件名

## 4.3 模块三：PIN 解锁

### 4.3.1 解锁机制

进入聊天前必须输入 PIN code。

### 4.3.2 解锁流程

```text
输入 PIN
  ↓
生成密钥（KDF）
  ↓
解密会话密钥
  ↓
解密聊天记录
  ↓
成功展示聊天
失败提示错误
```

### 4.3.3 错误策略

* 连续错误 5 次触发锁定策略
* 可配置清空该会话本地数据

## 4.4 模块四：聊天消息

### 4.4.1 文本消息

* 发送 -> 加密 -> 存储 -> 发送

### 4.4.2 图片消息

* 图片压缩/标准化 -> 加密 -> 上传 -> 消息引用

### 4.4.3 文件消息

* 支持发送通用文件附件
* 支持在聊天页展示文件名、大小、下载
* 对可预览文件类型支持预览动作

### 4.4.4 消息展示

* 解密后渲染
* 明文仅存在于当前会话内存与渲染态

## 4.5 模块五：联系人搜索与添加

### 4.5.1 搜索范围

* 支持按昵称搜索
* 支持按用户 ID 搜索
* 不支持按邮箱搜索

### 4.5.2 添加状态

* 搜索结果需展示已添加 / 可添加状态
* 重复添加应幂等

### 4.5.3 最近添加

* 页面展示最近添加联系人列表
* 默认按添加时间倒序

## 4.6 模块六：移动端适配

* 聊天列表 / 聊天页 / 搜索页在移动端切换为单列布局
* 聊天列表页提供底部导航
* 交互区域适配拇指操作

## 4.7 模块七：安全控制

### 4.7.1 自动锁定

* 切换标签页自动锁
* 超时自动锁

### 4.7.2 一键隐藏

* 清空 UI
* 返回运势页

# 6. 接口设计（产品口径）

## 6.1 搜索用户

搜索结果返回：

* userId
* nickname
* matchType
* alreadyAdded

不返回：

* 邮箱

## 6.2 会话列表

列表返回：

* 对端信息
* 未读数
* 最近消息时间
* 脱敏预览
* 在线状态摘要

## 6.3 附件消息

支持类型：

* 图片
* 通用文件

附件元数据至少包括：

* fileName
* fileExt
* fileCategory
* fileSize
* previewable
* downloadUrl

## 6.4 伪装页内容

运势接口至少返回：

* title
* summary
* fortuneScore
* luckyColor
* luckyNumber（用户输入值，非 luckyCode）
* suggestedActions
* adviceList
* moodReminder
* keywords

# 7. 非功能需求

## 7.1 安全

* luckyCode 允许在后端数据库明文存储，用于后端校验
* PIN 不存明文
* 会话列表真实内容脱敏
* 使用 KDF
* 使用随机 IV

## 7.2 性能

* 聊天列表加载 < 300ms
* 会话搜索为本地过滤时即时响应
* 单个常规文件卡片渲染流畅

## 7.3 兼容性

* Chrome / Safari / Edge
* 移动端浏览器
* 常见移动端竖屏布局

# 8. 风险与限制

## 8.1 PIN 被暴力破解

* 本地数据可能被导出

应对：

* 提升 KDF 成本
* 限制错误次数

## 8.2 浏览器安全问题

* XSS 可读取内存态明文

## 8.3 本地存储不绝对安全

* 用户设备被 root / 越狱后风险上升

## 8.4 用户遗忘 PIN

* 数据不可恢复

## 8.5 伪装被识破

* UI 或 URL 暴露真实用途

## 8.6 文件能力带来的额外风险

* 文件类型校验不足可能扩大攻击面
* 文件下载链路需要权限校验
