# PRD v1.2：隐私伪装聊天 Web 应用

## 1. 产品概述

### 1.1 产品定位

一个支持以下能力的 Web 隐私聊天工具：

* 伪装入口（幸运数字）
* 邮箱注册、登录、找回密码
* 联系人搜索与添加
* 1V1 文本、图片、通用文件附件消息
* 浏览器本地缓存消息记录
* 手机与 PC 浏览器自适应

### 1.2 当前版本核心能力

* 幸运数字作为隐藏入口
* 认证完成后直接进入聊天系统
* 会话列表脱敏展示
* 本地仅保留必要缓存，例如 `message_local`
* 刷新页面后如本地登录态仍有效，可继续直接进入聊天

### 1.3 不采用的机制

当前版本不采用以下机制：

* PIN 设置 / 输入 / 解锁
* PIN -> KEK -> conversationKey
* 会话级解锁
* 自动锁定后重新输入 PIN

## 2. 用户流程

```text
打开网页
  ↓
展示幸运数字输入页
  ↓
输入幸运数字
  ↓
通过后端接口校验 luckyCode
  ↓
命中 luckyCode：进入认证页或直接进入聊天系统
未命中 luckyCode：展示运势页
  ↓
完成邮箱注册 / 登录 / 找回密码
  ↓
进入聊天列表
  ↓
搜索联系人或进入添加好友页
  ↓
点击某个会话
  ↓
直接进入聊天界面并继续发送文本 / 图片 / 文件
```

## 3. 页面信息架构

### 3.1 幸运数字输入页

页面目标：

* 作为伪装入口第一层
* 输入幸运数字，不暴露聊天系统语义

页面元素：

* 顶部伪装文案
* 幸运数字输入框
* 进入按钮

交互要求：

* 幸运数字由后端 `luckyCode` 校验
* 前端不做本地幸运数字校验
* 校验失败不提示“密码错误”

### 3.2 运势展示页

页面目标：

* 提供完整伪装内容，降低聊天系统暴露感

### 3.3 聊天列表页

页面目标：

* 展示最近会话与联系人入口
* 支持搜索会话
* 支持移动端底部导航

列表展示字段：

* 对端昵称 / 备注名
* 最近消息时间
* 未读数
* 在线态摘要
* 脱敏预览

脱敏预览策略：

* 不允许展示真实消息文本
* 不允许展示真实文件名
* 统一展示固定占位

### 3.4 聊天页

页面目标：

* 认证完成后可直接进入聊天页
* 支持文本、图片、文件消息发送
* 历史消息无需再次解锁

### 3.5 搜索 / 添加好友页

页面目标：

* 支持按昵称、用户 ID 搜索用户
* 支持添加联系人
* 支持展示最近添加列表

展示限制：

* 不返回、不展示用户邮箱

## 4. 功能模块设计

### 4.1 模块一：幸运数字入口

* 后端数据库预置 `luckyCode` 作为隐藏入口校验值
* 用户输入后由后端校验
* 未命中则继续停留在伪装链路并展示运势页

### 4.2 模块二：认证

* 支持邮箱注册
* 支持邮箱密码登录
* 支持邮箱验证码登录
* 支持找回密码
* 认证成功后直接进入聊天系统

### 4.3 模块三：聊天消息

#### 4.3.1 文本消息

* 发送 -> 存储 -> 展示

#### 4.3.2 图片消息

* 图片上传 -> 消息引用 -> 聊天页展示

#### 4.3.3 文件消息

* 支持发送通用文件附件
* 聊天页展示文件名、大小、下载

#### 4.3.4 历史消息

* 历史消息可直接查看
* 可优先从本地 `message_local` 缓存恢复
* 不要求 PIN、会话密钥或解锁材料

### 4.4 模块四：联系人搜索与添加

* 支持按昵称搜索
* 支持按用户 ID 搜索
* 不支持按邮箱搜索
* 重复添加应幂等

### 4.5 模块五：移动端适配

* 聊天列表 / 聊天页 / 搜索页在移动端切换为单列布局
* 聊天列表页提供底部导航

### 4.6 模块六：安全控制

* 幸运数字仅承担伪装入口职责
* 会话列表保持脱敏
* 本地仅保留必要缓存
* 不引入新的替代性安全机制

## 6. 接口设计（产品口径）

### 6.1 搜索用户

搜索结果返回：

* userId
* nickname
* matchType
* alreadyAdded

不返回：

* 邮箱

### 6.2 会话列表

列表返回：

* 对端信息
* 未读数
* 最近消息时间
* 脱敏预览
* 在线状态摘要

### 6.3 附件消息

支持类型：

* 图片
* 通用文件

### 6.4 伪装页内容

运势接口至少返回：

* title
* summary
* luckyColor
* luckyNumber（用户输入值，非 `luckyCode`）
* adviceList

## 7. 非功能需求

### 7.1 安全

* `luckyCode` 允许在后端数据库明文存储，用于后端校验
* 会话列表真实内容脱敏
* 本地消息缓存不依赖 PIN 解锁

### 7.2 性能

* 聊天列表加载 < 300ms
* 会话搜索即时响应

### 7.3 兼容性

* Chrome / Safari / Edge
* 移动端浏览器

---

## 8. 风险与限制

### 8.1 浏览器安全问题

* XSS 仍可能读取当前页面内存态消息

### 8.2 本地存储不绝对安全

* 用户设备被 root / 越狱后风险上升

### 8.3 伪装被识破

* UI 或 URL 暴露真实用途

### 8.4 文件能力带来的额外风险

* 文件类型校验不足可能扩大攻击面
* 文件下载链路需要权限校验
