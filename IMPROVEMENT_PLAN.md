# HideChat 项目完善计划

基于 docs/、mockup/ 和 AGENTS.md 的分析，制定以下完善计划：

## 第一阶段：完善伪装入口系统（1-2天）

### 目标
实现完整的幸运数字验证和运势页显示功能

### 具体任务
1. **后端完善**
   - 完善 `SystemService` 中的幸运数字验证逻辑
   - 添加运势内容生成接口
   - 添加相应的 Controller 方法
   - 完善数据库 migration (V2__add_disguise_lucky_code.sql)

2. **前端完善**
   - 创建幸运数字验证页面组件
   - 创建运势显示页面组件
   - 实现页面路由切换逻辑
   - 添加相应的 API 调用

3. **测试完善**
   - 添加伪装入口的单元测试
   - 添加集成测试
   - 添加端到端测试

## 第二阶段：完善前端加密存储（1-2天）

### 目标
实现完整的 PIN 解锁和消息本地加密存储

### 具体任务
1. **PIN 解锁流程**
   - 完善 PIN 设置界面
   - 实现 PIN 验证逻辑
   - 添加自动锁定机制
   - 实现会话恢复流程

2. **本地加密存储**
   - 完善消息加密/解密逻辑
   - 实现 IndexedDB 存储封装
   - 添加存储清理策略
   - 实现离线消息缓存

3. **安全增强**
   - 添加 PIN 尝试次数限制
   - 实现安全擦除机制
   - 添加加密密钥轮换逻辑

## 第三阶段：完善测试覆盖（1天）

### 目标
确保关键路径都有充分的测试覆盖

### 具体任务
1. **后端测试**
   - 完善伪装入口测试
   - 添加加密相关测试
   - 完善集成测试套件

2. **前端测试**
   - 完善加密逻辑单元测试
   - 添加存储操作测试
   - 完善端到端测试

3. **性能测试**
   - 添加消息加密性能测试
   - 添加存储操作性能测试
   - 添加并发访问测试

## 第四阶段：文档同步和优化（0.5天）

### 目标
确保文档与实现保持一致

### 具体任务
1. **技术文档更新**
   - 更新技术方案文档
   - 完善 API 文档
   - 更新部署文档

2. **开发文档更新**
   - 更新 AGENTS.md（如果需要）
   - 完善 README.md
   - 添加开发指南

## 第五阶段：前端原型实现（1-2天）

### 目标
将 mockup 原型转换为实际的 React 组件

### 具体任务
1. **组件转换**
   - 运势页组件
   - 幸运数字页组件
   - 聊天列表组件
   - 聊天窗口组件
   - 添加好友组件

2. **样式优化**
   - 响应式设计优化
   - 移动端适配
   - 主题样式统一

## 实施优先级

1. **高优先级**：伪装入口系统（核心功能）
2. **高优先级**：PIN 解锁和加密存储（安全核心）
3. **中优先级**：测试覆盖（质量保证）
4. **中优先级**：前端原型实现（用户体验）
5. **低优先级**：文档同步（维护性）

## 风险控制

1. **不破坏现有功能**：所有修改都要确保现有功能正常
2. **逐步实施**：每个阶段完成后进行验证
3. **充分测试**：每个功能都要有相应的测试
4. **代码审查**：重要修改需要代码审查
5. **回滚计划**：每个阶段都要有回滚方案

## 成功标准

1. 伪装入口功能完整可用
2. PIN 解锁和加密存储功能完整可用
3. 关键路径测试覆盖率 > 80%
4. 文档与实现保持一致
5. 前端原型完整转换为可运行组件
