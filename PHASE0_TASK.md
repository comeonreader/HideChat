# Phase 0: 按照高保真UI设计修改当前产品UI

## 目标
将当前前端UI按照mockup高保真原型图进行修改，确保UI设计与设计文档一致。

## 范围
1. **幸运数字输入页**：按照`mockup/lucky-number.html`设计
2. **运势展示页**：按照`mockup/fortune.html`设计  
3. **聊天列表页**：按照`mockup/chat-list.html`设计
4. **聊天页**：按照`mockup/chat.html`设计
5. **搜索/添加好友页**：按照`mockup/add-friend.html`设计

## 设计规范
- **颜色方案**：使用mockup/styles.css中的CSS变量
- **布局结构**：遵循mockup的HTML结构
- **响应式**：支持桌面端双栏布局和移动端单列布局
- **组件样式**：按钮、卡片、输入框等组件样式保持一致

## 当前前端结构
- 主文件：`frontend/src/app/App.tsx`
- 样式文件：`frontend/src/app/app.css`
- 当前使用暗色主题，需要改为mockup的亮色主题

## 需要修改的内容

### 1. 样式系统更新
- 将`app.css`中的`:root`变量替换为mockup的CSS变量
- 更新全局样式以匹配mockup设计
- 确保响应式布局工作正常

### 2. 幸运数字输入页
- 更新UI以匹配`lucky-number.html`
- 包含顶部徽章、标题、副标题、输入框、按钮
- 保持幸运数字校验功能

### 3. 运势展示页
- 更新UI以匹配`fortune.html`
- 包含运势标题、整体走向、幸运色/数字/宜做、今日建议、情绪提醒
- 保持后端API数据加载功能

### 4. 聊天列表页
- 更新UI以匹配`chat-list.html`
- 桌面端：左侧边栏+主面板布局
- 移动端：底部导航
- 会话列表脱敏预览（必须显示`[文本消息]`等占位）

### 5. 聊天页
- 更新UI以匹配`chat.html`
- 左侧会话列表+右侧聊天区域
- 消息气泡样式
- 文件卡片样式
- 输入工具栏

### 6. 搜索/添加好友页
- 更新UI以匹配`add-friend.html`
- 搜索框+搜索结果+最近添加列表
- 桌面端双栏布局，移动端单列

## 技术要求
1. **保持功能完整性**：所有现有功能（幸运数字校验、PIN解锁、消息发送等）必须继续工作
2. **代码结构清晰**：合理组织组件，避免过度嵌套
3. **类型安全**：保持TypeScript类型定义
4. **响应式设计**：确保在手机和桌面浏览器都能正常显示

## 验证标准
1. 视觉上与mockup页面基本一致
2. 所有交互功能正常工作
3. 响应式布局正确
4. 会话列表脱敏策略正确实施（不显示真实消息）
5. 代码编译通过，无类型错误
