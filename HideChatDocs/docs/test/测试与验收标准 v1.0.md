适用范围：

* Web 端隐私聊天系统
* 后端：Spring Boot
* 前端：Web（手机浏览器 + PC 浏览器）
* 当前版本范围：邮箱登录、幸运数字伪装入口、联系人、1V1 会话、文本/图片消息、PIN 解锁本地历史消息

# 1. 文档目标

本文档用于统一：

* 功能测试范围
* 接口测试范围

# 2. 验收原则

本项目验收遵循 4 条原则：

## 2.1 核心链路优先

优先保证以下链路稳定：

* 注册 / 登录 / 找回密码
* 进入聊天系统
* 联系人列表
* 创建会话
* 收发消息
* 历史消息展示
* 图片上传与展示
* 会话未读数
* 自动锁定 / 隐藏

## 2.2 隐私边界要验证

虽然 Web 不是强安全容器，但必须验证：

* 幸运数字不明文存储
* PIN 不明文存储
* 本地历史消息默认不可直接读
* 页面锁定逻辑有效

## 2.3 前后端一致性

要验证：

* REST 接口正确
* WebSocket 消息状态正确
* 前端 IndexedDB 状态与后端会话索引基本一致

## 2.4 MVP 不追求完美，但必须可用

允许：

* 个别低优先级视觉问题
* 非核心功能的弱提示

不允许：

* 登录失败率高
* 消息丢失
* 未读数严重错误
* 本地密文失效
* PIN 解锁错误
* 图片消息不可用

# 6. 本地存储与隐私测试标准

---

# 6.1 浏览器本地数据测试

需要检查 IndexedDB / localStorage / sessionStorage：

### 必测项

* 幸运数字是否明文存储
* PIN 是否明文存储
* 历史消息是否明文存储
* `conversationKey` 是否明文持久化
* 页面刷新后锁定状态是否正确

### 通过标准

* 幸运数字：仅 hash / salt / KDF 参数
* PIN：仅 verifier / salt
* 消息：密文存储
* 图片：密文 Blob 存储

---

## 6.2 隐私场景测试

### 场景 A：别人拿到已解锁电脑但页面已锁

* 不能直接查看聊天内容

### 场景 B：别人打开聊天列表

* 只能看到聊过的用户，不应直接看到历史明文

### 场景 C：切到后台再切回

* 应要求重新输入 PIN 或符合锁定策略

### 场景 D：开发者工具查看本地存储

* 不应直接看到历史消息明文和 PIN 明文

# 12. 上线前最终验收标准

满足以下条件，才可判定 **MVP 可上线**：

## 必须全部满足

1. P0 用例 100% 通过
2. P1 用例通过率 ≥ 95%
3. 不存在阻断级安全漏洞
4. 不存在核心消息链路丢消息问题
5. 幸运数字、PIN、历史消息未明文落地
6. 主流浏览器兼容通过
7. 登录、聊天、图片发送、解锁、锁定五条主链路全部打通

## 可接受但需记录的问题

1. 少量 P2 交互问题
2. 文案与视觉细节问题
3. 极端弱网下体验抖动
4. 非主流浏览器轻微兼容问题

# 13. 建议的测试执行顺序

建议按下面顺序执行：

## 第一阶段：接口冒烟

* Auth
* User
* Contact
* Conversation
* Message
* File

## 第二阶段：主链路联调

* 注册/登录
* 进入聊天系统
* 添加联系人
* 创建会话
* 发文本
* 发图片
* 拉历史
* PIN 解锁

## 第三阶段：异常与安全

* token 过期
* 越权访问
* 错误验证码
* 重置密码
* 非法文件
* 频控

## 第四阶段：稳定性与兼容性

* 长连
* 重连
* 多浏览器
* 移动端
